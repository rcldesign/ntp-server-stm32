Time 101

First — how does anything
on the internet know the time?

Every phone, laptop, server and camera keeps time with a tiny quartz crystal — the same kind in a cheap wristwatch. It's good, not great: left alone it drifts, gaining or losing seconds a day.

So devices quietly phone home. A protocol called NTP (Network Time Protocol) lets a device ask a time server "what time is it, really?" and nudge its own clock to match. Those servers form a chain of trust called stratum — the closer to the top, the closer to true time.

The catch: that question travels across the internet, and the network adds unpredictable delay — typically 10–50 milliseconds of slop. Fine for your calendar. Not fine for correlating security logs, timestamping trades, telecom, industrial control, distributed databases, or any system where "when, exactly?" actually matters.

A millisecond is 1,000,000 nanoseconds. This box serves time good to a handful of nanoseconds — roughly a million times tighter than asking a server across the internet.

What it is

A stratum-1 time server you own —
not a server you borrow across the internet.

Instead of trusting a machine somewhere on the public internet, the STS-1000 gets time directly from GPS and Galileo satellites (which carry atomic clocks), disciplines a precision oven-controlled oscillator to that signal, and serves exact time to your whole network locally — microseconds over your LAN instead of milliseconds over the internet. And if the satellites drop out, the oscillator keeps holding accurate time on its own. One box, one cable, on your bench or in your rack.

How it works

Receive · Discipline · Serve

01

Receive from space

A timing-grade GNSS receiver locks onto GPS, Galileo, GLONASS and BeiDou and recovers UTC accurate to a few nanoseconds — plus a once-per-second pulse (PPS) aligned to the top of every second.

02

Discipline the oscillator

That pulse steers an OCXO — an oven-controlled crystal oscillator, a temperature-stabilized clock far steadier than any PC crystal — through a control loop, so the local clock stays locked to satellite time. (An optional rubidium atomic reference takes it further.)

03

Serve your network

The disciplined clock answers NTP, secure NTS and PTP requests with hardware-timestamped precision — for thousands of clients at once, over a single Power-over-Ethernet cable.

▲ HOLDOVER

Lose the sky? The OCXO coasts, holding sub-microsecond time for hours (days with the rubidium option) — and the box tells you exactly how far it might have drifted, second by second, so you're never guessing.

What you get

Carrier-grade timekeeping,
built for people who like to know how things work.

Stratum-1, from orbit

Time direct from GPS + Galileo + GLONASS + BeiDou. No upstream server to trust, no internet path to add error.

~Nanosecond accuracy

Phase offset to UTC measured in single-digit nanoseconds when locked — about a million times tighter than internet NTP.

Holdover that lasts

The OCXO keeps serving accurate time through GPS outages. Add a rubidium reference and that stretches into days.

Time you can trust

NTS (authenticated NTP) cryptographically signs the time it serves — so a client knows it's the real server, not a spoof.

PTP grandmaster

IEEE-1588 PTP for the networks that need sub-microsecond sync — finance, telecom, broadcast, automation.

Monitoring built in

SNMP exposes every metric — locks, offsets, temps, power, holdover — and drops straight into Zabbix.

Two ways to drive it

A rich local display with button navigation, plus a full HTTPS web console for config, calibration and updates.

Live satellite sky view

See exactly which satellites it's tracking, where they are overhead, and how strong each signal is — in real time.

Locked down

Hardware secure boot and signed firmware: only code you trust runs, and you control the keys.

The interface

Everything it knows, on the glass.

Time accuracy, the discipline loop, the satellite sky view, Allan-deviation stability plots, the event log, holdover predictions, network and security — the same data is on the device's display and in the web console.
